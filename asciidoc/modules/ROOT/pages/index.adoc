== History

Easykube started out as a collection of Gradle build scripts. The primary use case was to enable mobile developers to install real backends locally, allowing them to work offline and not require a personal instance of a test system on some random cluster.

The fontender's fancy MacBooks were powerful enough to drive a small Kubernetes instance, even with multiple backends, so why have their machines sit idle ?

Later, easykube was reimplemented in Go. And here we are :)
